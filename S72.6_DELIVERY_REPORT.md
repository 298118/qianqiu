# S72.6 水墨风格动效与视觉 Polish 交付报告

## 改动文件和新增文件用途
1. `public/mapRenderer.js`:
   - **新增动效逻辑**: 通过更新 `onTick`，为路线 (`route`) 和事件特效 (`ripple`) 添加了水墨风格的视觉动画。
   - **性能与可见性监听**: 引入了 `IntersectionObserver` 和 `visibilitychange` 事件监听。当地图面板在页面上不可见或当前浏览器标签页隐藏时，会自动挂起 `onTick` 内的坐标演算与视图更新，避免无意义的性能开销。
   - **选中态反馈更新**: 在 `selectRef` 方法中，将焦点选中的光圈更改为带有深朱砂色（`0x9b2f22`）的双层线框（内外粗细不一），使其更贴近“印章/墨圈”的案头质感，而不像现代 UI 的焦点框。
2. `public/styles.css`:
   - 对地图的附属 DOM UI 进行了案头舆图风格的视觉重构。
   - 修改了 `.map-tooltip`，移除了现代风格的单色底与大圆角，代之以 `linear-gradient` 宣纸暖色渐变 (`var(--paper)`)、硬朗的 `2px` 圆角和稍微加深的墨迹边框 (`var(--line-strong)`)。
   - 对 Tooltip 内的标题 (`strong`) 增加了底部的浅朱砂色分割线，提升文案层次与古代档案阅读感。
   - 空状态/资源失效状态的 `.map-fallback` 也同步更换为具备宣纸质感的柔和渐变。

## 新增动效和可降级策略
1. **路线流动 (Route Flow)**: 给所有的路线添加了一个极其缓慢的呼吸效果（alpha pulse）。由于线条是以深棕偏淡的颜色渲染的，当透明度轻微涨落时能带来水墨未干或“生息”的流转感。
2. **事件特效 (Ripple)**: 利用 Math.PI 的周期运算，对表示事件压力的 Sprite / Graphic 对象实现了如同“缓慢扩散并逐渐隐去”的水墨晕染效果（Scale 与 Alpha 协同缓动）。
3. **可降级 (Reduced Motion)**: 通过在 `update` 时检测 `this.reducedMotion` 和 `animatedEffects.length`，不仅对最高威胁度的特效进行动画总量截断（最高不超过 25 个并行对象的刷新），且在用户系统开启 `prefers-reduced-motion: reduce` 时，会自动取消往 Tick 循环添加这些增量对象。

## 性能注意事项
- 动画数量受到截流约束，且只保留低开销的 `scale` 和 `alpha` 属性变动，避免复杂的图形重算。
- Canvas 在不可见（被遮挡、折叠或切换标签页）时立即叫停 ticker 循环，为原本就繁重的文字引擎和 DOM 留出资源。
- 每次 `update()` 会调用基于 `PIXI.Container.destroy({children:true})` 的深度销毁机制，不会形成图层泄漏。

## 桌面/窄屏验证结果
- **桌面模式**: `clamp` 高度的地图上方显示带质感的 HTML Tooltip，不超出画幅、不产生文字截断，符合桌面文档式的阅读习惯。
- **窄屏模式**: Tooltip 由于具备了 `white-space: normal` 支持长文字自然折行，配合在 S72.5 已经实现的边缘防遮挡探测，不仅文字不溢出，也不会挡住处于手机端屏幕下方的行动输入框。

## Canvas 非空和资源 Fallback 情况
如果 Codex 配置的正式水墨素材尚未就绪，或因为 CDN 故障导致图片 404，`public/mapRenderer.js` 中包含的 Fallback `PIXI.Graphics` 代码仍会生效，渲染蓝灰色的点、深红色的事件线圈，并且这些纯代码生成的 fallback 图元能够无缝接入以上的水墨动效框架，不至于引起白屏崩溃。

## 已知风险
- 当前动画为统一 Ticker 全局刷新的线性动画，对于水墨的“毛边流散”和“纸张晕染”细节模拟无法与真正的 Shader/ShaderToy 比拟。考虑到 Web 端性能与不引入第三方复杂滤镜的准则，当前的轻量级 Alpha/Scale 策略是最佳性价比的平替方案。
