## Current Snapshot

- Product: browser + Node.js historical simulation text game **Qianqiu / 千秋**。
- Runtime target: `npm install && npm start`，然后打开 `http://localhost:3000`。
- Frontend: plain HTML/CSS/JS，无 build step。
- Backend: Node.js + Express，plain JavaScript。
- AI providers: adapter-based Mock/OpenAI/DeepSeek/MiMo/MiMo+DeepSeek/Anthropic。`AI_PROVIDER=mock` 仍是默认可玩模式，即使本机存在真实 provider key 也不会自动改走真实模型；S70.8 起 provider facade 有 task-aware route policy，`mimo-deepseek` 下 narrator/科举出题仍优先 MiMo，domain_specialist/科举评卷和 critic/safety 在 DeepSeek key 可用时走 DeepSeek。
- Storage: 默认 JSON session files under `data/sessions/`；可选 `STORAGE_ADAPTER=sqlite` 使用本地 `schema_migrations`、`world_sessions`、audit tables、`geo_*`、`people_*`、`office_*`、`event_archive_index`、`prompt_retrieval_index` 和 `safe_search_index`，FTS5 可用时另有 `safe_search_fts` 本地镜像。SQLite 派生行只从 `world_sessions.world_state_json` 单向修复；raw business/audit rows 不是 route、prompt、browser、搜索或服务器裁决 truth source。
- Active roadmap: S49-S67 本地数据库与大世界内容已统一归档，S68-S69 科举、读书、评卷、榜单、同年座师、授官和 Provider/Mock 验收已归档，S70 AI prompt/tool/actor/多模型路由、AI 设置、月报、跳时、记忆、地图接口、provider AI-first smoke、JSON/SQLite parity 和归档已完成，见 [AI_ORCHESTRATION_ARCHIVE.md](AI_ORCHESTRATION_ARCHIVE.md)。S71.0 数据库 resolver 输入契约已固定在 [DATABASE_RESOLVER_INPUT_CONTRACT.md](DATABASE_RESOLVER_INPUT_CONTRACT.md)，S71.1 已新增只读 `resolverInputContext` 输入层，S71.2 已新增本地 SQLite migration/maintenance 初版，S71.3 已新增安全全文检索 / 本地搜索，S71.4 已新增 redacted player API 与本机开发诊断 API，S71.5 已新增财政与城市政策服务器 resolver，S71.6 已新增地方案件与刑名服务器 resolver，S71.7 已新增军务与外交服务器 resolver，S71.8 已新增压力驱动事件生成器；下一步进入 S71.9 多 actor 场景运行时。
- Current local `.env`: 可能含用户提供的 provider keys。`.env` 被 Git 忽略，不能打印或提交。

## Core Invariants

- 完整书生路径必须继续可用：`scholar -> child_exam -> provincial_exam -> metropolitan_exam -> palace_exam -> official`。
- AI 是《千秋》的核心世界引擎，不是可替换装饰。新增玩法、数据域、角色、官署、事件、面板或 prompt 检索时，必须设计 AI read scope、actor intelligence、tool permissions、proposal boundaries、server adjudication、audit records 和 Mock/no-key fallback。
- Provider 只能建议叙事、受限 `statePatch`、关系变化、评分 JSON、考试触发或身份受限领域工具 proposal；服务器拥有晋级、考试资格、作弊处罚、持久化、保护字段、官职任免、长期事件、时间推进、数据库写入和可见性过滤。
- AI JSON 必须先验证再应用；状态变化必须经过白名单、clamp 和 server-owned follow-up modules。
- 游戏规则、阈值、时间间隔、概率、UI caps、fixture 规模和 prompt budgets 等可调参数不得散落为 magic numbers；优先放入 `src/config/GameConfig.js` 或领域 `src/game/*Config.js`，写明单位、范围和默认意图。
- `GET /api/game/saves` 只返回脱敏 metadata；普通浏览器读档优先通过 `GET /api/game/player-state/:sessionId` 读取 redacted player state 和清洗后的 route views，`GET /api/game/state/:sessionId` 仅作为短期开发兼容快照。
- 本地可玩性不能依赖真实模型 key；有 key 的 provider 检查必须在缺 key 时受控 skip 或 fail。
- 每个 coherent change 必须更新本交接板和步骤台账，运行相关验证，并用 Git 提交。
- 项目内协作文档、路线图、交接记录、领域注释和玩家可见文案优先使用中文；代码标识符、API/协议名、第三方术语和命令输出可保留英文。

## Content Protection

- 当前范围只考虑本地 JSON/SQLite；不规划远程存档、账号体系、多人同步、云冲突解决或托管数据库。
- AI 可以通过身份受限领域工具提交 proposal / request-adjudication，但不能执行 SQL，不能直接写 canonical 状态、`geo_*`、`people_*`、`office_*`、`event_archive_index`、`prompt_retrieval_index`、`event_log` 或 `ai_change_proposals`，工具调用也不能被当作已经发生的世界事实。
- 浏览器和 prompt 只读服务器生成的 view / capped retrieval summary，不读 raw SQLite table、raw audit、provider proposal、完整 prompt、本地路径、密钥、hidden notes 或 hidden intent；S70.12 起游戏/考试 route response 内的兼容 `worldState` 也会剥离 raw `actorMemoryLedger` 与 `sessionSummary`。
- hidden NPC 私档、资产真数、未公开关系、未公开任所、密档事件链和隐藏情报真值不得回填当前 raw route `worldState`。如果后续要保存完整 hidden 私档，先设计 API redaction 与角色视野分层；S70.12 的记忆/摘要账本已先按该方向做 response redaction。
- S66.1 的 `retrievalContext.strategy` 是只读上下文编排元数据，不是权限层。普通/high profile 仍约束在 48/72 行与约 20,000/30,000 字符；AI 仍不得写数据库、裁决事件、任免、战和、财政结算或公开 hidden。
- S66.2 的 `informationPanelPageView` 是浏览器局势簿分页 projection，只从服务器 route views 与安全事件档案条目生成；它不读 raw SQLite table、raw audit、provider proposal、完整 prompt、本地路径、key、hidden notes 或 hidden intent。
- S71.3 的 `safeWorldSearchView` 是玩家可见本地搜索 projection，只从服务器 route views、案牍/军务/财赋报告、公开事件链、事件档案和角色可见传闻生成；JSON 模式即时搜索这些 view，SQLite 模式只读 `safe_search_index` / 可选 FTS5 镜像。结果只返回 capped snippet、source view/id、visibility、confidence、related refs 和 route view ref；不返回内部行、审计原文、provider 输出、完整 prompt、本地路径、key、hidden notes 或 hidden intent。
- S71.4 的 `GET /api/game/player-state/:sessionId` 返回 `server_player_visible_state_projection` envelope，只含 allowlist 玩家可见状态、metadata、redaction 边界和二次清洗 route views；浏览器读档与局势簿分页已改读该 route。`GET /api/dev/session-diagnostics/:sessionId` 默认关闭，production 强制关闭，只在 `ENABLE_DEV_DIAGNOSTICS=true`、远端地址为 loopback，且 Origin 为空或本机 loopback Origin 通过时返回 storage/counts/resolver/AI 摘要统计；不返回 raw state、raw audit、provider payload、prompt、本地路径、key、SQLite 原始行或 hidden ledger。
- S71.5 的 `cityPolicyResolver` 是服务器拥有的财政/城市政策裁决层。AI 只能通过 `city.propose_policy` 提交政策 proposal；`server.adjudicate_policy` 不进入模型工具列表或 provider schema。resolver 只读取 S71.1 `resolverInputContext` 与 S70 领域工具可见 evidence refs，校验 T3-T5 权限、城市政策工具组、辖区、证据领域、资源余量和 hidden/raw 污染，再由服务器应用受控 meters、玩家官声字段、公开事件和内部 `cityPolicyLedger`；玩家 state route 不暴露 ledger/audit，普通 domain tool runner 仍保持 pending-only。
- S71.6 的 `judicialCaseResolver` 是服务器拥有的地方案件/刑名裁决层。AI 只能通过 `judicial.propose_case_resolution` 提交刑名 proposal；`server.resolve_case` 不进入模型工具列表或 provider schema。resolver 只读取 S71.1 `resolverInputContext` 与 S70 领域工具可见 evidence refs，校验 T2-T5 权限、`judicial` 工具组、辖区、证据领域、证据数量/可信度、惩罚性处置必须有可见案牍证据、重大案件制度路径和 hidden/raw 污染，再由服务器应用受控治安/腐败/玩家刑名考成字段、公开案牍摘要和内部 `judicialCaseLedger`；公开 docket/event 只显示可见证据摘要，不公开 evidence ref、raw proposal 或 hidden 证据，玩家 state route 不暴露 ledger/audit，普通 domain tool runner 仍保持 pending-only。
- S71.7 的 `militaryDiplomacyResolver` 是服务器拥有的军务/外交裁决层。AI 只能通过 `military.propose_order` 或 `diplomacy.propose_move` 提交 proposal；`server.resolve_battle` / `server.apply_diplomacy` 不进入模型工具列表或 provider schema。resolver 只读取 S71.1 `resolverInputContext` 与 S70 领域工具可见 evidence refs，校验 T4-T5 权限、军务/外交工具组、证据领域/数量/可信度、低可信边报、钱粮/市场证据、兵粮资源、target scope、制度路径和 hidden/raw 污染，再由服务器应用受控府库/粮储/军心/边患/玩家将领或外交字段、公开事件和内部 `militaryDiplomacyLedger`；公开 event 只显示可见证据摘要，不公开 evidence ref、raw proposal 或 hidden 情报，玩家 state route 不暴露 ledger/audit，普通 domain tool runner 仍保持 pending-only。
- S71.8 的 `worldPressureEventGenerator` 是服务器拥有的压力成案层。它只从 S70.6 可见 pressure refs 与 S71.1 `resolverInputContext` 收集脱敏压力信号，按 `worldPressureEventConfig` 的组合规则、优先级、分数、确定性概率、冷却和每旬上限生成事件候选；`resolvePressureEventCandidate()` 只返回 outcome，`applyPressureEventOutcome()` 才写受控 world meter、公开 `eventHistory` 和内部 `worldPressureEventLedger`。公开事件只显示可见线索摘要，不公开 source ref、raw proposal、hidden 私档、密情真值或 server.* label；`worldPressureEventLedger` 不进入 redacted player state，S70.6 事件工具协议仍保持 proposal/pending 边界。

## Subagent Discipline

- 用户已授权 Codex 和 Claude Code 在本仓库使用子代理，除非后续指令撤销，否则视为长期项目上下文。
- 实施子代理只能做 scoped patches 和 focused verification；不得运行 `git add`、`git commit`、`git push` 或创建 PR。
- 主代理负责整合、最终验证、共享文档同步和唯一 coherent commit。
- 包含代码、测试、运行时行为、API/schema、提示词或验证工具变化的提交，暂存和提交前必须至少委派一个只读子代理审查最终 diff 与验证证据。
- 低风险纯文档改动可跳过子代理复审；路线图重写或内容安全边界变更即使是文档，也应做只读复审。
