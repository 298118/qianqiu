# S73.10.1 前端全量立绘矩阵

本文档固定 S73.10 全量玩家/NPC 立绘池的生产矩阵。机器可读版本见 `public/assets/ui/portraits/portrait-pool-matrix-v1.json`，生成与校验入口为 `npm run qa:portrait-matrix`。

S73.10.1 只定矩阵、命名、prompt 母版和审核字段，不批量生图，不把未生成素材写入 `ink-ui-manifest.json` 可用资产集合。S73.10.2 已按 player 分组完成首批 72 张玩家身份阶段立绘、缩略图、低清占位、QA sidecar 和 manifest 入库，并保留 60 张此前已审核女性玩家风格补充；S73.10.3 已完成通用 NPC 池 188 张入库，其中一开始的 120 张按矩阵保留，另有 20 张旧版 bonus 和 48 张宫装/唐装女性风格扩展。后续仍由 Codex 使用 `gpt-image-2` 生成、视觉审核、压缩、入库和登记。未通过审核的候选图不得进入 runtime usable 状态。

## 1. 数量与分层

目标总量锁定为 336 张，位于 S73.10 规划的 300-400 张区间内，便于按小批次生成、审核和回滚。

| 分层 | 数量 | 后续步骤 | 用途 |
| --- | ---: | --- | --- |
| 玩家身份阶段 | 72 | S73.10.2 | 男女玩家从书生、童试、秀才、举人、贡士、进士到地方官、京官、大臣、将领、皇帝/摄政的身份成长 |
| 通用 NPC | 120 | S73.10.3 | 已完成矩阵 120 张；另有 20 张旧版 bonus 与 48 张女性宫装/唐装风格扩展作为已审核通用补充池 |
| 重要 NPC 专属 | 72 | S73.10.4 | 皇帝、太后、摄政、首辅、御史、总督、名将、权臣、清流领袖、宿敌和知己等剧情核心模板 |
| 状态与姿态变体 | 48 | S73.10.5 | 沉思、拱手、执卷、阅卷、持笏、按剑、疲惫、病中、含怒、请罪等可复用状态池 |

## 3. Prompt 母版

以下为后续批量生图的创意母版。它们刻意保留画面感和角色气息，不写成参数清单；最终每批 prompt 仍由 Codex 定稿。

1. 成年书生立于泛黄宣纸前，青衫素净，袖中露出书卷一角，神情清朗而克制，像刚从长夜读书中抬眼；水墨淡彩，线条温润，半身立绘，背景极淡，不出现文字。
2. 成年女书生着月白襦裙与深青披帛，腰封收束，手持未题字的卷轴，眉目沉静，有读书人的锋芒也有端庄气度；宣纸肌理、水墨晕染、半身正侧构图。
3. 新科进士身着红袍礼服，簪花但不过分华丽，神情有喜色也有敬畏，身后只见淡淡榜墙与春风墨痕；高颜值、成年、端正，古代中国官服氛围，无可读文字。
4. 县令夜坐案前，乌纱与青绿官袍压住疲惫，手边是无字案卷和朱印，目光谨慎，像刚听完一桩难案；暖黄灯影、宣纸底、水墨人物立绘，半身，庄重不阴森。
5. 年长主考官端坐阅卷，须发整洁，目光严厉而不刻薄，手持朱笔但卷面空白不可读；构图留出上方纸雾，色调沉稳，体现科场法度与文脉威仪。
6. 成年女官身着深绛与墨青层叠官服，腰封、玉佩、宽袖形成清晰轮廓，姿态肃立，手持笏板，目光平静而有权柄；水墨淡彩，端庄成熟，不露骨，不幼态。
7. 边关将领披暗红披风与旧甲，手按佩剑，肩背挺直，脸上有风霜，不夸张战斗姿势；背景是淡墨军帐与风沙纸纹，半身立绘，历史感强，无现代军械。
8. 宫廷贵人立于屏风前，衣料层次细密，金色只作微光点缀，神情含蓄难测，既有礼制约束也有政治重量；宣纸、水墨、淡彩，不出现龙凤文字纹样，不做艳俗姿态。
9. 市井商贾着深褐长衫与短外袍，手中把玩算盘或账册但无字，笑意谨慎，像懂得风向的人；人物亲和但不滑稽，纸色背景，墨线清楚，适合 NPC 头像池。
10. 清流御史身形瘦削，持笏拱手，衣冠整肃，眼神锋利，像要在朝堂上直言进谏；冷灰宣纸底，一点朱砂印痕作气氛，不出现可读奏疏文字。
11. 幕友师爷侧身坐在书斋边，手握折扇或笔杆，衣着不显官阶但干净精明，神情温和里带盘算；淡墨书斋背景，半身立绘，留白多，适合对话面板。
12. 皇帝常服立绘，不做神化大场面，玄色与明黄克制搭配，目光疲惫而有威压，身后只是模糊御案与纸色屏风；成年、端庄、历史模拟游戏风格，无文字、水印或现代元素。

统一安全收束语：

角色必须是明确成年人，气质端庄、历史人物立绘风格；不做幼态脸、儿童或少年化身体比例，不做挑逗姿态、裸露、透视衣料或性暗示；画面不得出现现代物品、现代发型妆容、摄影棚感、logo、水印、签名、二维码、可读文字、伪汉字、乱码字、文件路径、API key、raw/hidden/prompt 泄漏性内容。

## 4. 防重复策略

最大风险是“清一色俊男美女半身站姿、同一张纸色背景、同一种冷淡表情”。后续批量生成时按以下维度轮换：

- 服饰：书生用棉麻青衫，官员用补服/常服/朝服差异，武将用旧甲与披风，商贾用暗纹长衫，宫廷人物用层叠礼服。
- 姿态：拱手、执卷、持笏、侧身回望、按剑、扶案、垂袖肃立、执笔停顿、袖手沉思轮换。
- 道具：书卷、笏板、朱笔、无字案卷、折扇、算盘、佩剑、玉佩、药箱、印匣、灯盏，少而准，不堆杂物。
- 色温：书生青灰，科举暖纸黄，朝堂冷灰朱砂，军务赭红烟灰，宫廷墨金，市井茶褐。
- 构图：主池保持半身统一，但混用正面、三分之二侧面、案前坐姿、肩部裁切和左右留白，保证缩略图轮廓不全都相同。

## 5. 安全与接线边界

- `portrait-pool-matrix-v1.json` 不是 manifest 可用资产。S74-S77 runtime 只能使用 `ink-ui-manifest.json` 中已审核、`runtimeUsable` 为 true 的立绘。
- 矩阵不保存完整 provider 原始响应，不保存本地绝对路径、key、raw audit、hidden notes、hidden intent、未公开任所、未公开关系或剧情密档。
- 后续每张成图必须经过 Codex 视觉理解审核，确认中国古代形象、水墨/淡彩一致性、身份辨识、小尺寸可读、无现代物、无水印、无乱码、无露骨/挑逗/幼态问题。
- 重要 NPC 的 `portraitRef` 可以先存在于矩阵，但不得因为矩阵存在就向玩家公开 hidden 私档或未公开剧情事实。
- 全量池不得首屏加载；React 资产层必须按 route、公开人物列表、当前身份和用户操作懒加载缩略图、低清占位或源图。
