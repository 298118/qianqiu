# S73.1 前端视觉资产指南

本文档固定 S73-S77 前端水墨重构的视觉资产标准。它服务于后续 S73.2 UI manifest、S73.3-S73.10 素材生成与全量立绘入库、S74 资产加载层和 S75-S77 多页前端验收；本步只立规范，不生成图片、不安装依赖、不改变运行时代码。

## 1. 总原则

- 阅读优先，装饰服从文本。任何纸纹、墨晕、边框、动效或人物图都不得降低叙事、考题、榜文、存档和设置文字的可读性。
- 资产必须像案头旧史书、展开画卷、奏折和册页，而不是现代仪表盘皮肤。
- 所有玩家可见文字由 HTML/CSS/JS 渲染；图片中不得生成可读文字、伪中文、英文标识、地图地名、榜名单字或签名水印。
- S73 资产只描述前端视觉，不新增后端事实、隐藏私档、AI 权限或服务器裁决。身份面板、人物谱牒、考试和舆图只能读取已有安全 route view 或未来明确新增的安全 projection。
- `mapRuntimeView` 继续只供浏览器地图 runtime 显示；地图显示坐标、图标和底图不进入 prompt、AI 工具、移动裁决、财政、战争、任免或 canonical 世界事实。
- S73-S77 正式 AI 生成素材统一由 Codex 使用 `gpt-image-2` 生成。所有 AI 生成和第三方候选资产入库前，必须由 Codex 做视觉审核并在后续素材台账记录结论。未审核或未通过的素材不得进入 manifest 的可用集合。

## 2. 目录与命名

S73.2 起建议使用以下目录；S73.1 不创建空目录：

```text
public/assets/ui/
  ink-ui-manifest.json
  materials/
  home/
  scenes/
  roles/
  portraits/
  effects/
  fallback/
  thumbs/
```

命名规则：

- 使用 `kebab-case-vN.ext`，例如 `paper-xuan-soft-v1.webp`、`portrait-scholar-male-young-official-v1.png`。
- 版本递增不覆盖旧文件。替换资产使用 `v2`，并在 manifest 和台账说明替换原因。
- 文件名只描述安全用途、风格和版本，不写真实 prompt、人物 hidden 身份、本地路径、provider 名称、key 片段或未公开剧情。
- `id` / `portraitRef` / `fallbackRef` 稳定引用资产，不在 React 组件、地图桥或其他前端运行时代码里硬编码具体图片路径。

推荐分类前缀：

| 分类 | 前缀 | 示例 |
| --- | --- | --- |
| UI 材质 | `material-` | `material-folded-memorial-v1.webp` |
| 首页 | `home-` | `home-scroll-landscape-v1.webp` |
| 场景插画 | `scene-` | `scene-exam-cell-night-v1.webp` |
| 身份背景 | `role-` | `role-magistrate-yamen-v1.webp` |
| 立绘 | `portrait-` | `portrait-minister-female-formal-v1.png` |
| 动效序列 | `effect-` | `effect-cinnabar-stamp-v1.webp` |
| fallback | `fallback-` | `fallback-paper-panel-v1.webp` |
| 缩略图 | `thumb-` | `thumb-scene-yamen-v1.webp` |

页面 usage 建议：

| usage | 对应页面 |
| --- | --- |
| `home` | `/` 首页画卷、开局、案卷式存档 |
| `game_main` | `/game/:sessionId` 主叙事页 |
| `map_page` | `/game/:sessionId/map` 独立舆图页 |
| `people_page` | `/game/:sessionId/people` 人物谱牒页 |
| `archive_page` | `/game/:sessionId/archive` 史册/局势簿页 |
| `exam_page` | `/game/:sessionId/exam` 科举考试页 |
| `ranking_page` | `/game/:sessionId/ranking` 放榜页 |
| `court_page` | `/game/:sessionId/court` 皇帝/朝议或官署公文页 |
| `settings_surface` | 右上角印匣、设置、存档和显示偏好 |

后续资产加载层应按 usage、role、scene 和可见 `portraitRef` 懒加载；不得在首页或主叙事页一次性加载 S73.10 全量立绘池、全部场景插画或全部专题背景。

## 3. 色彩与材质

基础色板：

| 用途 | 建议色 | 说明 |
| --- | --- | --- |
| 主纸色 | `#f5f0e6` | 宣纸米白，页面主底 |
| 旧绢色 | `#e8dcc8` | 画卷、册页、抽屉底 |
| 深墨 | `#241f18` | 正文、标题、重点文本 |
| 淡墨 | `#6f6659` | 次级文本、分隔线、注释 |
| 朱砂 | `#a53a2a` | 主行动、印章、榜名、警示 |
| 暗朱 | `#6f2119` | hover、选中圈、危险态 |
| 青黛 | `#2f5c5a` | 地图商路、低强度强调 |
| 官青 | `#34495e` | 大臣/官署次强调 |
| 低饱和金 | `#b08a47` | 皇帝、殿试、放榜少量点缀 |

材质原则：

- 宣纸和旧绢纹理必须低对比，正文区域叠加后仍满足长时间阅读。
- 奏折、皇榜、题名册、试卷界栏和抽屉背景可以有折痕、残纸边、骑缝章和浅墨线，但边缘不能抢正文。
- 朱砂只作为行动、选中、榜文和警示点缀，不能大面积铺满界面。
- 避免高饱和渐变、玻璃拟态、霓虹光效、现代卡片阴影和圆角过大的塑料感。

## 4. 字体与文字承载

- 图片不得承载正式 UI 文案。所有“千秋”、按钮、榜名、考题、评语、姓名、地名和设置说明都由 DOM 文本渲染。
- 背景图必须预留文字安全区，安全区内纹理更淡、对比更低、无复杂人物或建筑线条。
- 标题可使用更庄重的中文字体策略，但正文优先清晰的系统中文字体。后续若引入 Web font，必须走依赖/资源治理并提供 fallback。
- 固定格式 UI 要给文字留出稳定尺寸，不能让按钮、底部奏折、考试写作区、存档案卷因长名字或错误信息挤压变形。

## 5. 文件规格

推荐尺寸：

| 用途 | 建议源尺寸 | 比例 | 格式 | 备注 |
| --- | --- | --- | --- | --- |
| 首页主画卷 | 2560x1440 | 16:9 | WebP | 首屏主视觉，中心或上方留白 |
| 首页云雾层 | 2560x720 | 宽幅 | PNG/WebP | 低动效可静态替代 |
| 主叙事纸面 | 1600x1200 | 4:3 | WebP | 可平铺或九宫格裁切 |
| 场景插画 | 1920x1080 | 16:9 | WebP | 书斋、贡院、县衙、军帐等 |
| 身份背景 | 1800x1200 | 3:2 | WebP | 右侧或中央预留面板区 |
| 考试/放榜全屏 | 2400x1350 | 16:9 | WebP | 背景不可含可读榜文/试题 |
| UI 材质小图 | 512x512 或 1024x1024 | 方形 | WebP/PNG | 纸纹、折痕、边框 |
| 立绘源图 | 1024x1536 | 2:3 | PNG/WebP | 半身或七分身，透明优先 |
| 立绘 UI 缩略 | 384x576 | 2:3 | WebP/PNG | 列表/面板懒加载 |
| 图标/印章 | 512x512 | 方形 | PNG/WebP | 透明或纯色底 |
| 缩略图 | 320x180 或 256x256 | 用途相关 | WebP | manifest 预览和素材 QA |

压缩预算：

- 首页首屏关键背景单张目标不超过 1.5 MB。
- 场景和身份背景单张目标不超过 1.2 MB。
- UI 材质单张目标不超过 300 KB。
- 立绘源图可以较高质量保存，但首屏不得一次性加载 S73.10 全量立绘池；UI 缩略图目标不超过 250 KB。
- 动效优先用短序列、CSS transform、透明纹理或静态 fallback，避免大体积视频。

## 6. 裁切与响应式

每张背景资产必须在台账或 manifest 中记录 `safeArea` 和 `focalPoint`：

- `safeArea`：文字和表单可覆盖区域，建议用归一化矩形 `{ x, y, width, height }`。
- `focalPoint`：移动端裁切时应尽量保留的视觉中心。
- `mobileCrop`：说明移动端可以裁掉的边缘，不得裁掉角色头部、主建筑、表单留白或榜文容器。

裁切规则：

- 首页 16:9 背景在窄屏可裁左右，但必须保留标题留白、表单底和一点下方内容提示。
- 主叙事和设置抽屉背景宁可使用可平铺纸纹，也不要强行裁切复杂大图。
- 科举考试和放榜全屏背景必须给中央试卷/皇榜 DOM 留出低对比区域。
- 立绘必须有头顶和衣袖余量，透明边界不能贴脸、断手或裁掉官服关键身份符号。

## 7. UI 材质标准

S73.3 UI 材质包至少覆盖：

- 宣纸底：主页面、叙事卷和 fallback 面板。
- 旧绢底：首页表单、设置抽屉、存档案卷。
- 奏折/折子：底部行动输入、官署公文、皇帝朱批。
- 竹简/册页：辅助信息、人物摘要、读书簿。
- 试卷界栏：科举写作区和考题区。
- 皇榜纸底：放榜页面和名次列表。
- 淡墨分隔线：面板边界、tab、列表。
- 朱印按钮：开始、呈上、交卷、确认读档等主行动。
- 印匣纹理：右上角设置入口和抽屉头部。

材质必须支持纯 CSS 颜色 fallback。图片加载失败时，页面仍显示可读纸色、边框和按钮状态。

## 8. 首页与场景资产

首页主视觉：

- 第一视口必须直接传达“千秋”和古代史书/画卷气质。
- 背景应有山水、云雾、城郭、驿道或书案元素，但中央表单区必须留白。
- 不使用现代 landing page 式分屏卡片，不把主体验包在大圆角营销卡里。

场景插画：

- 书斋：清寒、竹影、旧书、砚台，服务书生路径。
- 贡院号舍：狭小、专注、紧张，不血腥不猎奇。
- 放榜：热闹但不拥挤，榜上无可读文字。
- 殿试/御案：庄重、克制，少量金色，不堆满龙纹。
- 县衙/公堂：制度感、案牍、庭院，避免夸张刑具。
- 军帐/边塞：军务紧张但不暴力猎奇。
- 城市街巷：生活气、商旅、城门，可用于地图或人物页背景。

每张场景图必须记录用途、允许出现的路由、留白位置、fallback 和安全审核状态。

## 9. 身份背景

身份背景不代表 hidden 状态，只代表玩家当前可见身份和场景氛围。

| 身份 | 视觉关键词 | 避免 |
| --- | --- | --- |
| 书生 | 书斋、竹影、寒窗、旧册、清志 | 华丽宫廷、直接暗示必中功名 |
| 地方官 | 县衙、公堂、案牍、钱粮、水利 | 夸张刑罚、血腥、现代徽章 |
| 入仕官员 | 部院文书、观政、任所、同年 | 直接伪造升迁事实 |
| 大臣 | 内阁/部院、公文、朝议、考成 | 过度皇帝化、直接暗示任免 |
| 将领 | 军帐、粮草、斥候、边关 | 前端直接表现已胜战果 |
| 皇帝 | 御案、奏折、玉玺、殿阁 | 强金光、现代帝王幻想、直接写圣旨结果 |

身份背景只给 UI 气质和入口分组，不得让前端绕过服务器行动、resolver 或普通回合裁决。

## 10. 立绘规范

所有玩家与 NPC 立绘必须满足：

- 角色为成年人，姿态端正，表情克制，身份明确。
- 古代服饰、发冠、饰物和器物要符合身份气质；不得出现现代妆造、现代饰品、现代军装、现代办公用品或品牌。
- 高颜值可以来自五官、神情、气质、服饰层次和构图；不得来自露骨、挑逗、幼态化或身体部位卖点。
- 女性角色可以是女官、才女、商贾、宫廷人物、将领、市井人物或亲族人物。可用服饰剪裁、腰封、衣料层次和站姿体现优雅成熟女性身形比例，但衣着必须端庄，不能过度暴露、湿身、低胸、挑逗姿势或未成年感。
- S73.10.2a 已批准的 60 张玩家女性风格补充池是玩家可选立绘，不归入 NPC 池；后续选角界面可通过 `player_female_style_pool`、`portrait_pool_player_female_extra_s73_10` 和已审核 `portraitRef` 按需加载，重点补宫装、唐装、女官、才女、商贾、边关将领等服饰/姿势变化。
- S73.10.2b 已批准的 60 张玩家男性风格补充池是玩家可选立绘，不归入 NPC 池；后续选角界面可通过 `player_male_style_pool`、`portrait_pool_player_male_extra_s73_10` 和已审核 `portraitRef` 按需加载，重点补书院文士、唐风圆领袍、朝堂官服、宫廷贵胄、中枢重臣、边塞将领、禁军武官、士族公子、商贾东家和行旅策士等服饰/姿势变化。
- S73.10.3 已批准的女性风格扩展池可作为后续通用立绘补充使用：`palace-lady` 与 `tang-lady` 通过 `female_style_pack`、`palace`、`tang` 标签检索，重点补宫装与唐装的服饰/姿势变化；这些图是 336 张锁定矩阵之外的额外已审核通用池，不替代原本 120 张通用 NPC 矩阵。
- 男性角色同样要求端庄、身份清楚，可通过肩颈与上身轮廓、冠帽、腰带、甲衣、胡须和姿态表现成年男性气质；不得使用过度肌肉化、裸露、现代偶像造型或夸张动漫脸。
- 背景透明或极淡宣纸底，方便叠加在人物谱牒、身份面板和对话 surface。
- 立绘不得含可读文字、水印、签名、真实人物肖像相似性、hidden 私档暗示、未公开关系或密情真值。

立绘裁切：

- 标准 UI 使用半身或七分身，头部、肩部、衣领、腰封和身份器物完整。
- 面板展示建议保留透明安全边距，避免 hover 边框或纸面遮住发冠。
- 列表缩略图优先裁头肩，仍保留身份符号；没有合适缩略图时用身份剪影 fallback。

立绘全量生产与使用：

- S73.7 先完成 24-40 张风格基准，验证审美、安全、裁切、manifest、fallback 和 UI 适配。
- S73.10 在 S73 内完成 300-400 张全量玩家/NPC 立绘池，包括玩家身份阶段、常见 NPC、女官和女性 NPC、重要历史/拟史人物、姿态状态变体、缩略图和低清占位。
- 已审核的额外补充立绘可以在不改变 336 张矩阵目标的前提下进入 manifest，例如 S73.10.2a 的 60 张玩家女性风格补充、S73.10.2b 的 60 张玩家男性风格补充、S73.10.3 的 20 张旧版 bonus 与 48 张宫装/唐装女性风格扩展；补充资产必须有独立标签、QA sidecar、视觉/安全审核和懒加载组。
- 每张立绘必须在 manifest 和台账中登记 `portraitRef`、usage、role、gender、ageBand、statusVariant、thumbnail、fallback、reviewStatus、safeArea 和 focalPoint。
- S74-S77 的人物显示只能通过已审核 `portraitRef`、缩略图和 fallback 使用；不得硬编码图片路径，不得显示未审核素材，不得让重要 NPC 混用通用头像池。
- S73.10 全量立绘池不得首屏全量加载；React 资产层必须按 route、角色、公开人物列表和可见 `portraitRef` 懒加载。

## 11. 动效与低动效

允许的动效：

- 云雾慢移、淡墨呼吸、纸页轻展、朱印落章、墨迹扩散、榜文揭示、火漆轻压。
- 地图沿用 S72 路线墨线呼吸和事件涟漪，但必须尊重 `prefers-reduced-motion`。
- 页面切换可以用短墨晕淡入或纸页展开，不使用强弹跳、旋转、闪烁和现代粒子爆炸。

动效预算：

- 单个交互反馈建议 120-280ms。
- 场景氛围循环必须缓慢、低对比，不吸走阅读注意力。
- `prefers-reduced-motion: reduce` 或用户显示偏好关闭动效时，必须使用静态图、透明度变化或无动画状态。
- 动效失败不得阻断开局、读档、提交行动、考试交卷或查看榜单。

## 12. Fallback 规则

每类资产都必须有 fallback：

| 资产 | fallback |
| --- | --- |
| 首页背景 | 纸色底 + 淡墨边 + DOM 标题 |
| 场景插画 | 对应身份纸纹或通用宣纸底 |
| 身份背景 | 角色身份色 + 淡墨分隔线 |
| 立绘 | 身份剪影、印章头像或纸面占位 |
| 动效 | 静态最后帧或无动画状态 |
| 图标/按钮 | CSS 朱印按钮、文字按钮和可聚焦边框 |
| 舆图独立页 | S72 纸色静态底与文字列表入口 |

fallback 不得只是空白。即使所有图片加载失败，玩家仍应能开局、读档、提交行动、考试、放榜和回到首页。

## 13. 审核清单

每个 AI 生成或第三方候选素材入库前必须检查：

- 风格：是否符合《千秋》水墨、手绘、宣纸、奏折、旧史书和古代历史模拟气质。
- 阅读：是否有足够留白，叠加正文、表单、榜单、考题或按钮后仍清楚。
- 历史：是否避免现代建筑、现代服装、现代 UI、现代地图符号和不合身份器物。
- 安全：是否无可读乱码、水印、品牌、本地路径、key、raw table、hidden notes、hidden intent、未公开关系或密档真值。
- 人物：是否成年、端庄、身份明确、非露骨、非挑逗、非幼态。
- 技术：尺寸、比例、格式、透明度、alpha 边缘、文件大小和缩略图是否符合用途。
- 一致性：色彩、笔触、颗粒、边缘和明暗是否能与同批资产共存。
- 可维护：是否有稳定 id、版本、fallback、台账记录、替换策略和审核状态。

审核状态建议：

- `draft`：草案或未处理候选，不可用于运行时。
- `review_pending`：等待 Codex 视觉审核，不可默认展示。
- `approved`：可进入 manifest 可用集合。
- `approved_with_limits`：只可用于特定路由、尺寸或背景。
- `rejected`：不得进入 manifest；台账记录拒绝原因。
- `replaced`：已被新版替换，保留历史记录。

## 14. Manifest 与台账边界

S73.2 的 `ink-ui-manifest.json` 只保存浏览器需要的安全字段，例如：

- `id`、`version`、`category`、`usage`、`role`、`scene`
- `path`、`thumbnailPath`、`fallbackRef`
- `dimensions`、`aspectRatio`、`format`、`transparent`
- `safeArea`、`focalPoint`、`mobileCrop`
- `motion`、`reducedMotionFallback`
- `reviewStatus`、`ledgerId`

manifest 不保存：

- 完整 prompt 原文。
- provider 原始响应。
- `.env`、key、base URL、本地绝对路径。
- raw audit、raw SQLite table、hidden notes、hidden intent。
- 未公开 NPC 私档、资产真数、密档事件链或隐藏情报真值。
- 第三方长许可证文本。许可证和来源细节进入 `docs/FRONTEND_ASSET_LEDGER.md`。

第三方来源在台账中必须区分 `reference-only`、`texture-source` 和 `direct-asset-candidate`。参考图不等于可入库资产；直接资产候选必须先确认许可，再通过 Codex 视觉审核。
