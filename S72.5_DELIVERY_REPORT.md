# S72.5 地图与现有游戏 UI 联动 交付报告

> Codex 复审补记：本文件保留 Gemini CLI 原始交付说明。最终入库实现以本轮 Git diff、`docs/DEVELOPMENT_STEPS.md` 和 `docs/SHARED_CONTEXT.md` 为准；Codex 后续将动态 `querySelector` 改为 dataset 安全匹配，隐藏 tab 跳转改为不触发 `app.js` 重渲染的 DOM 内 tab 激活，并补充事件 hitArea 与异步素材重绘修正。

## 改动文件和新增文件用途
1. `public/mapRenderer.js`:
   - 新增了 `selectRef(ref, pos)` 接口。
   - 在各图层对象（路线 `routes`、地点 `places`、事件特效 `events`）的 `pointertap` 事件回调中，调用 `selectRef` 以便通过 `PIXI.Graphics` 在 `layers.selection` 容器中绘制高亮光圈，从而在点击地点、路线或事件 marker 后，更新地图选中态。
2. `public/mapPanel.js`:
   - 修改了 `handleClickRef(ref, pos)` 逻辑。除了展示原本安全的 `ref.label` 和 `ref.summary` 外，还加入了到局势簿的联动机制。
   - 增加对 Tooltip 位置超出父容器的保护（Edge Boundary Detection），防止窄屏遮盖或溢出。

## 接入的现有 DOM / app.js 函数
1. **获取信息面板卡片**: 使用了原生的 DOM Selector `document.querySelector('article[data-entity-id="..."]')`。
2. **定位面板 Tab**: 沿用现有 `information-panel-page` 及其 ID，并在获取 Tab DOM 后使用 `click()` 模拟玩家点击，以此实现面板的切换。
3. **滚动并高亮**: 使用了 `card.scrollIntoView()` 并在该卡片上附带两秒钟的 CSS 翡翠色 `box-shadow` 作为视觉强调。
4. **行动输入框**: 获取 `#action-input`，并将 `actionText` 设置为其值。

## 地图点击如何联动 UI
当玩家在画布中点击某个对象：
1. `mapRenderer` 捕捉到 `pointertap`，通过 `selectRef` 清除上一个选中对象并在该坐标画出一个选中的高亮圈。
2. `mapRenderer` 会向外 emit 到 `mapPanel.js` 注册的 `onClickRef` 监听器。
3. `mapPanel` 清除旧的 tooltip，在安全限制下展示名称和 summary。
4. 此时检测 `ref.sourceRef` 或者 `ref.mapEntityRef`。如果在底下的局势簿中可以 query 到该 ID 所对应的 `<article>` DOM（即已经在公开局势中存在），则在气泡中多渲染一个“查阅局势簿”按钮。
5. 玩家点击查阅按钮，页面会自动翻转至对应 Tab 并滚动高亮到那张细目卡片。如果未被公开渲染，则仅能在提示气泡看到安全的 summary 摘要。

## Action draft 如何保持服务器裁决边界
地图气泡里展示的行动选项完全来自于服务器提供的安全映射 `actionDraftRefs`。前端仅能解析 `actionDraft.actionText`，点击“行动草拟”时前端仅仅是将这段自然语言文字写入了 `#action-input` 输入框中（`input.value = draft.actionText;`），然后关闭弹窗。
它**绝不会自动提交**，也无法绕过系统修改游戏进程，玩家仍必须手动确认文字后点击“行动”按钮走常规 `POST /api/game/turn` 解析。

## 桌面/窄屏验证结果
- **桌面模式**: `map-panel` 按响应式限制占据一定高度，高亮圈精确指示焦点，tooltip 内容不溢出，联动可顺利滚动面板。
- **窄屏模式**: Tooltip 进行了 `getBoundingClientRect` 的边缘裁切反转检测，当提示框超过地图层左、右、上边界时会通过计算自行推回安全区，交互完全不遮挡下方的行动输入框。

## 未覆盖的风险或需要 Codex 后端补充的字段
- 目前对于部分无法通过 `data-entity-id` 被 DOM 搜索到的长尾事件或特殊路线，只能依赖 Tooltip。如果 Codex 后期扩充局势簿或新增卡片类型，其对应 ID 也应保持与 `mapEntityRef` 后缀一致以便自动挂钩。
- 对于通过不同 Tab 展示同一 entity 的场景，目前代码就近找到第一个匹配的面板。这是受限于无前端路由的单页状态所能做到的最稳定安全联动。
